# 第二章：计算机内部的数据表示

## 3 定点数的编码

### 3.2 补码

#### 一、补数

以某个数为模，与负数等价的正数称为负数的补数。

例如，对于模为 12 的系统，-3 的补数是 9（因为 -3 ≡ 9 mod 12）。补数的核心思想是将减法转化为加法，从而简化运算。

#### 二、定点整数的补码表示

对于定点整数，补码的定义如下（假设字长为 n 位，包含一位符号位）：

- **正数**：补码等于其本身（即原码）。
- **负数**：补码等于 \(2^n\) 加上该负数（即 \(2^n - |x|\)）。

用数学公式表达为：

$$
\[
[x]_{\text{补}} = 
\begin{cases} 
x, & 0 \le x < 2^{n-1} \\[1ex]
2^n + x, & -2^{n-1} \le x < 0 
\end{cases}
\]
$$

其中 \(x\) 为真值（十进制整数），\(n\) 为总位数。

例如，8 位定点整数（n=8）：
- \(+5\) 的补码：\(00000101\)
- \(-5\) 的补码：\(2^8 - 5 = 256 - 5 = 251\)，二进制为 \(11111011\)。

#### 三、定点小数的补码表示

定点小数通常将小数点固定在最高位之后（纯小数）。对于 n 位定点小数（包含一位符号位），补码定义为：

\[
[x]_{\text{补}} = 
\begin{cases} 
x, & 0 \le x < 1 \\[1ex]
2 + x, & -1 \le x < 0 
\end{cases}
\]

注意：此处 \(x\) 是真值（十进制小数），\(2 + x\) 相当于对负数加 2（模 2）。-1 的补码表示为 \(1.000\cdots0\)（n 位，符号位为 1，数值位全 0）。

例如，8 位定点小数（1 位符号位，7 位数值位）：
- \(+0.5\) 的补码：\(0.1000000\)
- \(-0.5\) 的补码：\(2 + (-0.5) = 1.5\)，二进制小数 \(1.1000000\)。

#### 四、补码的转换公式

从原码或真值求补码的简便方法：

1. **正数**：补码与原码相同。
2. **负数**：
   - 方法一：根据定义 \(2^n + x\)（整数）或 \(2 + x\)（小数）直接计算。
   - 方法二：对负数的绝对值（正数）的原码**按位取反（包括符号位）再加 1**。注意：对于整数，取反加 1 后得到的是补码的数值部分，符号位自然为 1；对于小数，取反加 1 同样适用（加法需考虑进位）。

例如，8 位整数 -5：
- 绝对值 5 的原码：\(00000101\)
- 取反：\(11111010\)
- 加 1：\(11111011\)（得到补码）。

**补码的补码**：对补码再次求补（即取反加 1）会得到原码。

#### 五、IEEE754 浮点数标准（补充内容）

IEEE754 是浮点数表示的国际标准，定义了单精度（32 位）和双精度（64 位）浮点数的格式。其表示形式为：

\[
(-1)^S \times M \times 2^{E - \text{bias}}
\]

其中：
- **S**：符号位（0 表示正，1 表示负）。
- **E**：阶码（exponent），用移码（biased representation）表示。
- **M**：尾数（mantissa），用原码表示，且隐含整数部分为 1（规格化数）。

**单精度浮点数（32 位）**：
- 符号位 S：1 位
- 阶码 E：8 位，偏置常数 bias = 127
- 尾数 M：23 位，表示小数部分，整数部分隐含为 1（即尾数实际为 \(1.M\)）。

**双精度浮点数（64 位）**：
- 符号位 S：1 位
- 阶码 E：11 位，偏置常数 bias = 1023
- 尾数 M：52 位。

**特殊值**：
- 阶码全 0 且尾数全 0：表示 \(\pm 0\)。
- 阶码全 1 且尾数全 0：表示 \(\pm \infty\)。
- 阶码全 1 且尾数非 0：表示 NaN（Not a Number）。

例如，单精度浮点数 1.0 的表示：
- 二进制：\(1.0 = 1.0 \times 2^0\)，S=0，E=127（移码），M=0（隐含 1），故 32 位为：0 01111111 00000000000000000000000。

#### 六、补码的优点（补充内容）

补码在现代计算机中广泛使用，主要优点如下：

1. **统一加减法**：补码可以将减法运算转化为加法运算，只需加法器即可完成加减法，大大简化硬件设计。
2. **0 的表示唯一**：补码中 0 只有一种表示（例如 8 位补码 00000000），避免了原码中 \(+0\) 和 \(-0\) 的歧义。
3. **符号位直接参与运算**：补码的符号位可以和数值部分一起参与运算，结果自动正确，无需额外判断符号。
4. **便于溢出判断**：补码运算中，可以通过最高位进位与次高位进位的异或来判断是否溢出（即“双高位法”）。
5. **表示范围对称**：补码能表示的最小负数比原码多一个（例如 8 位补码范围为 \(-128 \sim +127\)，而原码为 \(-127 \sim +127\)），更充分地利用了位数。


#### 补码的缺点
- **理解难度较高**：相比于原码，补码的转换规则不够直观。
- **乘除法不直观**：符号位需额外处理，不如原码直接。
- **溢出判断复杂**：需要检测符号位变化或进位情况。



#### C语言中采用补码的数据类型
C语言标准（C20起明确要求）规定所有有符号整数类型均使用**补码**表示，包括：
- `signed char`
- `short`
- `int`
- `long`
- `long long`

无符号整数类型（如 `unsigned int`）采用直接二进制编码，不涉及补码。  
浮点类型（`float`, `double`等）通常遵循 IEEE754 标准，其内部表示并非补码。



### 3.3 反码（One's Complement）

#### 一、反码的基本概念

反码是计算机中一种有符号数的二进制表示方法，主要作为原码到补码的过渡。其定义规则如下：

- **正数**：反码与原码相同。
- **负数**：符号位保持不变（仍为1），数值位按位取反（0变1，1变0）。

例如，对于8位二进制数：
- 正数 +5 的原码为 `00000101`，其反码也为 `00000101`。
- 负数 -5 的原码为 `10000101`，其反码为 `11111010`。

#### 二、定点整数的反码

假设定点整数字长为 n 位（包含一位符号位），则反码的定义可以用数学公式表示：

\[
[x]_{\text{反}} = 
\begin{cases} 
x, & 0 \le x < 2^{n-1} \\[1ex]
(2^{n} - 1) + x, & - (2^{n-1} - 1) \le x < 0
\end{cases}
\]

其中 \(x\) 为真值（十进制整数），\(n\) 为总位数。  
对于负数，\( (2^{n} - 1) + x \) 实际上等价于对绝对值的原码按位取反（因为 \(2^{n} - 1\) 是全1的二进制数）。

**示例**（n=8）：
- \(x = +5\)：\([+5]_{\text{反}} = 00000101\)。
- \(x = -5\)：\([-5]_{\text{反}} = (2^8 - 1) - 5 = 255 - 5 = 250\)，二进制为 \(11111010\)。

**数值范围**：n 位定点整数的反码能表示的数值范围为 \(-(2^{n-1} - 1) \sim +(2^{n-1} - 1)\)。例如8位反码的范围是 \(-127 \sim +127\)。

#### 三、定点小数的反码

对于定点小数（纯小数，小数点固定在符号位之后），假设字长为 n 位（包含一位符号位），反码定义为：

\[
[x]_{\text{反}} = 
\begin{cases} 
x, & 0 \le x < 1 \\[1ex]
(2 - 2^{-(n-1)}) + x, & -1 < x \le 0
\end{cases}
\]

其中 \(x\) 为真值（十进制小数），\(2 - 2^{-(n-1)}\) 是 n 位二进制小数的最大值（即全1）。负数反码等价于符号位不变，数值位按位取反。

**示例**（n=8，1位符号位+7位数值位）：
- \(x = +0.5\)：二进制原码为 \(0.1000000\)，反码相同 \(0.1000000\)。
- \(x = -0.5\)：原码为 \(1.1000000\)，反码为 \(1.0111111\)（符号位1，数值位取反）。

**数值范围**：n 位定点小数的反码能表示的数值范围为 \(-(1 - 2^{-(n-1)}) \sim +(1 - 2^{-(n-1)})\)。

#### 四、反码的优缺点

##### 优点
1. **与补码的过渡**：反码是理解补码的基础，负数补码可通过反码加1得到。
2. **运算逻辑简单**：取反操作容易通过硬件实现（非门）。
3. **对称性**：正负数的表示范围对称（除0外）。

##### 缺点
1. **零的表示不唯一**：反码中存在 \(+0\) 和 \(-0\) 两种表示。例如8位反码中，\(+0 = 00000000\)，\(-0 = 11111111\)。这给运算和判断带来歧义。
2. **运算需循环进位**：反码做加法时，若最高位产生进位，需要将进位加到最低位（循环进位），增加了硬件复杂度。
3. **数值范围比补码少一个**：由于零占用了两种表示，可表示的数值个数比补码少一个（补码中零唯一，多出一个最小负数）。

由于以上缺点，现代计算机系统普遍采用补码表示有符号整数，而反码仅作为理论过渡或特定场景（如某些校验算法）使用。
